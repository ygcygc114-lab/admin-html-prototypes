3. 背景

当前视频管理需要支持一条内容出现在多个菜单位置。原来偏向“一条内容只关联一个菜单”的逻辑，无法满足同一内容同时投放到多个菜单的运营需求。

4. 目标

  • 支持一条内容关联多个菜单。
  • 列表中的“关联菜单”字段清楚展示该内容已关联的菜单集合。
  • 内容是否展示由内容整体上架状态统一控制，降低运营理解成本。
  • 设置菜单弹窗与列表字段联动，便于演示完整操作闭环。

5. 核心概念

概念说明
内容视频列表中的单条视频内容。
菜单前台展示内容的导航或分类位置，支持多级结构。
关联菜单某条内容已经绑定的一个菜单节点。
菜单路径从一级菜单到目标菜单的完整路径，例如：一级菜单A / 二级菜单B / 三级菜单C / 四级菜单D。
内容状态内容整体的上架状态，决定内容是否在已关联菜单下展示。

关联到菜单仅在“内容已上架 + 当前菜单未关联”时展示的关联按钮，含义等同于“仅关联”。

取消关联解除内容和某个菜单的绑定关系。

6. 业务规则

6.1 内容与菜单关系

  1. 一条内容可以关联多个菜单。
  2. 同一条内容不可重复关联同一个菜单路径。
  3. 菜单关联关系只记录内容绑定了哪些菜单，不记录菜单维度上下架状态。
  4. 内容已上架后，会在所有已关联菜单下展示；内容未上架时，所有菜单均不展示。
  5. 内容已上架但没有任何关联菜单时，内容状态仍为已上架，但前台没有菜单入口展示。
  6. 关联并上架 更新的是内容整体状态，因此会影响该内容所有已关联菜单的展示。
  7. 关联到菜单 的前置条件是内容整体状态已上架，且目标末级菜单当前未关联；如果内容未上架，不展示该按钮。
内容菜单路径内容状态
内容A一级菜单A / 二级菜单A / 三级菜单B已上架
内容A一级菜单A / 二级菜单B / 三级菜单C / 四级菜单D已上架

6.2 菜单可关联规则

  1. 父级菜单不允许关联内容，只用于层级展示。
  2. 判断标准不是菜单级数，而是该菜单是否有子菜单。
  3. 有子菜单的节点统一视为父级菜单，操作列显示“无法关联”。
  4. 没有子菜单的末级菜单才允许关联内容。

6.3 状态流转规则

操作展示前置判断：系统先判断目标菜单是否可关联，再判断内容是否已关联该菜单，最后判断内容整体状态。只有“可关联末级菜单 + 未关联 + 内容已上架”时展示“关联到菜单”。

操作结果
仅关联新增菜单关联，不改变内容状态。
关联并上架新增菜单关联，并将内容整体状态更新为已上架。
内容已上架 + 当前菜单未关联只展示“关联到菜单”，关联后会在该菜单下展示。
未上架内容关联新菜单展示“仅关联 / 关联并上架”。
取消关联删除该菜单路径下的关联关系。

6.4 多选规则

  1. 多选内容时，操作会应用到所有已选内容。
  2. 如果选中内容里存在未上架内容，未关联末级菜单展示“仅关联 / 关联并上架”。
  3. 如果选中内容全部已上架，未关联末级菜单只展示“关联到菜单”。
  4. 点击 仅关联 时，所有已选内容新增菜单关联，内容状态保持原状。
  5. 点击 关联并上架 时，所有已选内容新增菜单关联，并统一更新为已上架。
  6. 点击 取消关联 时，所有已选内容删除该菜单关联，内容状态保持原状。

8. 页面需求

8.1 视频列表 - 关联菜单字段

列表中的“关联菜单”字段展示该内容当前已关联的菜单集合。每条关联展示菜单路径和目标菜单名称，不展示菜单维度上下架状态。

  • 默认最多展示 2 条菜单关联。
  • 如果关联菜单数量大于 2 条，展示 +N 查看全部。
  • 如果关联菜单数量小于或等于 2 条，不展示“查看全部”。
  • 未关联任何菜单时，显示 未关联。

8.2 查看全部弹窗

点击 +N 查看全部 后，展示该内容全部已关联菜单。弹窗字段包括菜单路径，不展示“列表展示”等内部说明文案。

8.3 设置菜单入口

  • 入口位于视频列表批量操作区域中的 设置菜单 按钮。
  • 未勾选内容时，按钮置灰不可点击。
  • 勾选至少 1 条内容后，按钮高亮可点击。
  • 支持单选和多选内容后批量设置菜单。

8.4 设置菜单弹窗

设置菜单弹窗按菜单树展示。父级菜单显示“无法关联”；未上架内容的未关联末级菜单显示“仅关联 / 关联并上架”；只有“内容已上架 + 当前菜单未关联”时，才显示“关联到菜单”。

  • 弹窗顶部需要说明：菜单只表示内容展示位置；内容是否展示由内容状态统一控制。
  • 弹窗顶部需要说明：关联并上架会将内容整体状态更新为已上架，并在所有已关联菜单下展示。

8.5 批量绑定菜单弹窗

  • 点击 批量绑定菜单 后打开弹窗。
  • 弹窗展示内容数量输入框、当前项目、目标菜单树。
  • 内容数量最大值为 1000。
  • 弹窗规则说明需要展示当前已选内容的上架状态判断，说明为什么展示“关联到菜单”或“仅关联 / 关联并上架”。
  • 如果已选内容全部已上架，未关联末级菜单只展示“关联到菜单”。
  • 如果已选内容里存在未上架内容，或未明确选择内容，末级菜单保留“仅关联 / 关联并上架”。
  • 点击“仅关联”表示只新增菜单关联，不改变内容状态。
  • 点击“关联并上架”表示新增菜单关联，并将命中的内容整体状态更新为已上架。
  • 弹窗内需要展示批量规则说明，避免用户误认为“关联并上架”是菜单维度状态。

9. 联动规则

  1. 打开设置菜单弹窗时，弹窗根据当前选中内容的已关联菜单展示对应状态。
  2. 点击 仅关联 或 关联到菜单 后，列表中该内容的“关联菜单”字段立即新增该菜单，内容状态保持不变。
  3. 点击 关联并上架 后，列表中该内容的“关联菜单”字段立即新增该菜单，并将内容整体状态更新为已上架。
  4. 点击 取消关联 后，列表中该内容的“关联菜单”字段立即移除该菜单。
  5. 若当前选中了多条内容，操作应用到所有已选内容。

10. 筛选逻辑

  1. 内容只要关联了筛选条件对应的菜单路径，即命中筛选。
  2. 如果内容关联了多个菜单，筛选不同菜单时，同一条内容可能在多个结果中出现。
  3. 筛选后，列表中的“关联菜单”字段仍展示该内容全部已关联菜单，而不是只展示当前筛选命中的菜单。
  4. 为降低“内容重复”的误解，当前筛选命中的菜单需要展示“当前筛选命中”标记。
  5. 筛选项中的“内容状态”代表内容整体上下架状态。
  6. 菜单筛选与内容状态筛选取交集，不存在“当前菜单状态”。

11. 验收标准

11.1 列表展示

  • 内容无关联菜单时，显示“未关联”。
  • 内容关联 1 条菜单时，展示 1 条菜单路径。
  • 内容关联 2 条菜单时，展示 2 条菜单路径，不展示“查看全部”。
  • 内容关联超过 2 条菜单时，展示前 2 条，并显示 +N 查看全部。
  • 点击“查看全部”后，弹窗展示全部菜单路径。
  • 按菜单筛选后，命中的关联菜单展示“当前筛选命中”标记。
  • 筛选项保留“内容状态”和菜单筛选，结果取交集。

11.2 设置菜单弹窗

  • 未勾选内容时，设置菜单按钮不可点击。
  • 勾选内容后，设置菜单按钮高亮可点击。
  • 弹窗菜单按层级树展示。
  • 父级菜单显示“无法关联”。
  • 未上架内容的未关联末级菜单显示“仅关联”和“关联并上架”。
  • 只有“内容已上架 + 当前菜单未关联”时，未关联末级菜单才显示“关联到菜单”。
  • 已关联菜单展示“已关联”和“取消关联”。
  • 批量绑定菜单弹窗需要展示当前已选内容状态判断，避免用户误以为按钮没有更新。

11.3 联动效果

  • 点击“仅关联”或“关联到菜单”后，列表字段新增对应菜单，内容状态保持不变。
  • 点击“关联并上架”后，列表字段新增对应菜单，并将内容状态更新为已上架。
  • 点击“取消关联”后，列表字段移除对应菜单。
  • 多选内容时，操作对所有已选内容生效。
  • 多选内容包含未上架内容时，设置菜单弹窗展示“仅关联 / 关联并上架”。
  • “关联并上架”后，该内容在所有已关联菜单下展示。
  • 内容已上架但无关联菜单时，内容状态保持已上架，但前台无菜单入口展示。
