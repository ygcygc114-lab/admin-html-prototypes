## 2. 本次新增/变更内容

本次主要是在原“视频管理”页面上新增内容与菜单的多关联能力，并补齐对应的列表展示、弹窗操作和筛选联动规则。

| 新增/变更项 | 用户能看到什么 | 说明 |
| --- | --- | --- |
| 内容支持关联多个菜单 | 同一条内容的“关联菜单”字段可展示多个菜单路径 | 原逻辑偏向一条内容只绑定一个菜单，本次改为一条内容可绑定多个菜单。 |
| 关联菜单字段改造 | 列表默认展示前 2 条关联菜单，超过 2 条显示 `+N 查看全部` | 点击“查看全部”后，可查看该内容已关联的全部菜单和状态。 |
| 菜单维度状态 | 每个关联菜单单独展示已上架/未上架 | 同一内容在菜单A可以已上架，在菜单B可以未上架。 |
| 设置菜单弹窗 | 弹窗按菜单树展示，末级菜单可操作 | 父级菜单只做层级展示，显示“无法关联”；末级菜单支持“仅关联”“关联并上架”“取消关联”。 |
| 批量绑定菜单弹窗 | 显示内容数量、当前项目、目标菜单树 | 菜单树和操作状态与“设置菜单”弹窗保持一致。 |
| 筛选命中标记 | 按菜单筛选后，命中的关联菜单展示“当前筛选命中” | 列表仍展示该内容全部关联菜单，避免用户误以为内容只绑定了当前筛选菜单。 |
| PRD入口 | 原型右上角新增 `PRD` 按钮 | 方便评审、开发、测试直接在原型内查看需求说明。 |

## 3. 背景

当前视频管理中，内容列表支持对视频进行菜单关联、上架、下架等操作。原逻辑偏向“一条内容只关联一个菜单”，无法满足同一内容出现在多个菜单位置的运营需求。

新需求希望支持：

1. 一条内容可以关联多个菜单。
2. 内容在不同菜单下可以有不同的上架状态。
3. 列表中的“关联菜单”字段能清楚展示该内容已关联的菜单情况。
4. “设置菜单”弹窗可以按菜单树选择具体菜单，并与列表展示联动，便于运营演示和真实使用。

## 4. 目标

### 4.1 产品目标

- 支持内容和多个菜单建立关联关系。
- 支持菜单维度的上架状态，而不是只按内容整体上架状态判断。
- 降低运营理解成本，让用户能快速知道一条内容绑定了哪些菜单、在哪些菜单下已上架或未上架。
- 让“设置菜单”弹窗和列表“关联菜单”字段形成一致的操作闭环。

### 4.2 不做事项

- 本期不调整内容本身的正片、预览、详情编辑能力。
- 本期不新增菜单管理能力，只消费已有菜单树。
- 本期不处理复杂权限策略，仅按现有后台权限体系展示入口。

## 5. 核心概念

| 概念 | 说明 |
| --- | --- |
| 内容 | 视频列表中的单条视频内容。 |
| 菜单 | 前台展示内容的导航/分类位置，支持多级结构。 |
| 关联菜单 | 某条内容已经绑定的一个菜单节点。 |
| 菜单路径 | 从一级菜单到目标菜单的完整路径，例如：`一级菜单A / 二级菜单B / 三级菜单C / 四级菜单D`。 |
| 菜单维度状态 | 同一条内容在某一个菜单下的状态，例如已上架、未上架。 |
| 仅关联 | 内容绑定到菜单，但不立即在该菜单下上架。 |
| 关联并上架 | 内容绑定到菜单，并在该菜单下立即上架。 |
| 取消关联 | 解除内容和某个菜单的绑定关系。 |

## 6. 业务规则

### 6.1 内容与菜单关系

1. 一条内容可以关联多个菜单。
2. 同一条内容不可重复关联同一个菜单路径。
3. 内容关联菜单后，需记录该菜单维度下的状态。
4. 内容在不同菜单下的状态互相独立。

示例：

| 内容 | 菜单路径 | 菜单维度状态 |
| --- | --- | --- |
| 内容A | 一级菜单A / 二级菜单A / 三级菜单B | 未上架 |
| 内容A | 一级菜单A / 二级菜单B / 三级菜单C / 四级菜单D | 已上架 |

### 6.2 菜单可关联规则

1. 父级菜单不允许关联内容，只用于层级展示。
2. 判断标准不是菜单级数，而是该菜单是否存在子菜单。
3. 有子菜单的节点统一视为父级菜单，操作列显示“无法关联”。
4. 没有子菜单的末级菜单才允许关联内容。

示例：

| 菜单节点 | 是否可关联 | 说明 |
| --- | --- | --- |
| 一级菜单A | 否 | 父级菜单 |
| 二级菜单A | 否 | 父级菜单 |
| 三级菜单A | 否 | 下方还有四级菜单 |
| 四级菜单A | 是 | 末级菜单 |
| 三级菜单B | 是 | 没有下级菜单 |

### 6.3 状态规则

| 操作 | 结果 |
| --- | --- |
| 仅关联 | 新增菜单关联，状态为未上架 |
| 关联并上架 | 新增菜单关联，状态为已上架 |
| 对已关联菜单再次选择“仅关联” | 保留关联，将状态更新为未上架 |
| 对已关联菜单再次选择“关联并上架” | 保留关联，将状态更新为已上架 |
| 取消关联 | 删除该菜单路径下的关联关系 |

## 7. 页面需求

### 7.1 视频列表 - 关联菜单字段

#### 展示内容

列表中的“关联菜单”字段展示该内容已关联的菜单集合。

每条菜单关联展示：

1. 菜单路径父级部分。
2. 目标菜单名称。
3. 当前菜单维度状态：已上架 / 未上架。

示例：

```text
一级菜单A / 二级菜单B / 三级菜单C
四级菜单D  已上架

一级菜单A / 二级菜单A
三级菜单B  未上架
```

#### 展示数量

1. 默认最多展示 2 条菜单关联。
2. 如果关联菜单数量大于 2 条，展示 `+N 查看全部`。
3. 如果关联菜单数量小于或等于 2 条，不展示“查看全部”。
4. 未关联任何菜单时，显示 `未关联`。

#### 查看全部弹窗

点击 `+N 查看全部` 后，展示该内容全部已关联菜单。

弹窗字段：

| 字段 | 说明 |
| --- | --- |
| 菜单路径 | 完整菜单路径 |
| 状态 | 当前内容在该菜单下的状态 |

弹窗不展示“列表展示”等内部说明文案。

### 7.2 设置菜单入口

#### 入口位置

视频列表批量操作区域中的 `设置菜单` 按钮。

#### 启用规则

1. 未勾选内容时，按钮置灰不可点击。
2. 勾选至少 1 条内容后，按钮高亮可点击。
3. 支持单选和多选内容后批量设置菜单。

### 7.3 设置菜单弹窗

#### 基础信息

弹窗标题：`设置菜单`

顶部字段：

| 字段 | 说明 |
| --- | --- |
| 项目名称 | 当前项目，例如 WX1-01 |

提示文案：

```text
以下操作只影响当前内容在对应菜单下的关联与上下架状态。
```

#### 菜单树展示

菜单列表需要按层级树展示。

示例：

```text
一级菜单A [一级菜单]          无法关联
  二级菜单A [二级菜单]        无法关联
    三级菜单A [三级菜单]      无法关联
      四级菜单A [四级菜单]    仅关联 / 关联并上架
      四级菜单B [四级菜单]    仅关联 / 关联并上架
    三级菜单B [三级菜单]      仅关联 / 关联并上架
  二级菜单B [二级菜单]        无法关联
    三级菜单C [三级菜单]      无法关联
      四级菜单D [四级菜单]    仅关联 / 关联并上架
```

#### 操作列状态

| 菜单状态 | 操作展示 |
| --- | --- |
| 不可关联父级菜单 | 无法关联 |
| 未关联末级菜单 | 仅关联 / 关联并上架 |
| 已关联未上架 | 已关联未上架 / 取消关联 |
| 已关联已上架 | 已关联已上架 / 取消关联 |

#### 弹窗和列表联动

1. 打开弹窗时，弹窗根据当前选中内容的已关联菜单展示对应状态。
2. 在弹窗中点击 `仅关联` 后，列表中该内容的“关联菜单”字段立即新增该菜单，状态为未上架。
3. 点击 `关联并上架` 后，列表中该内容的“关联菜单”字段立即新增该菜单，状态为已上架。
4. 点击 `取消关联` 后，列表中该内容的“关联菜单”字段立即移除该菜单。
5. 若当前选中了多条内容，操作应用到所有已选内容。

## 8. 筛选逻辑

### 8.1 菜单筛选

页面顶部支持按一级、二级、三级、四级菜单进行筛选。

筛选规则：

1. 内容只要关联了筛选条件对应的菜单路径，即命中筛选。
2. 若内容关联了多个菜单，筛选不同菜单时，同一条内容可能在多个结果中出现。
3. 列表中的“关联菜单”字段仍展示该内容全部已关联菜单，不只展示当前筛选命中的菜单。

原因：

- 该字段表达的是“内容已关联菜单集合”，不是“当前筛选命中菜单”。
- 如果筛选后只展示命中菜单，用户会误以为内容只关联了当前菜单。

命中标记：

- 为降低“内容重复”的误解，当前筛选命中的菜单需要展示“当前筛选命中”标记。
- 标记只说明该菜单命中了当前筛选条件，不改变内容和菜单的真实关联关系。

## 9. 交互流程

### 9.1 仅关联流程

1. 用户在列表勾选内容A。
2. `设置菜单` 按钮高亮。
3. 用户点击 `设置菜单`。
4. 弹窗展示菜单树。
5. 用户在末级菜单上点击 `仅关联`。
6. 系统新增内容A与该菜单的关联关系。
7. 该菜单状态为未上架。
8. 列表“关联菜单”字段同步显示该菜单和未上架状态。

### 9.2 关联并上架流程

1. 用户在列表勾选内容A。
2. 用户打开 `设置菜单` 弹窗。
3. 用户在末级菜单上点击 `关联并上架`。
4. 系统新增内容A与该菜单的关联关系。
5. 该菜单状态为已上架。
6. 列表“关联菜单”字段同步显示该菜单和已上架状态。

### 9.3 取消关联流程

1. 用户在列表勾选已关联菜单的内容。
2. 用户打开 `设置菜单` 弹窗。
3. 已关联菜单展示当前状态和 `取消关联` 按钮。
4. 用户点击 `取消关联`。
5. 系统删除该内容与该菜单的关联关系。
6. 列表“关联菜单”字段同步移除该菜单。
7. 若该内容没有任何关联菜单，列表显示 `未关联`。

## 11. 验收标准

### 11.1 列表展示

- [ ] 内容无关联菜单时，显示 `未关联`。
- [ ] 内容关联 1 条菜单时，展示 1 条菜单路径和状态。
- [ ] 内容关联 2 条菜单时，展示 2 条菜单路径和状态，不展示“查看全部”。
- [ ] 内容关联超过 2 条菜单时，展示前 2 条，并显示 `+N 查看全部`。
- [ ] 点击 `查看全部` 后，弹窗展示全部菜单路径和状态。
- [ ] 按菜单筛选后，命中的关联菜单展示“当前筛选命中”标记。

### 11.2 设置菜单弹窗

- [ ] 未勾选内容时，`设置菜单` 按钮不可点击。
- [ ] 勾选内容后，`设置菜单` 按钮高亮可点击。
- [ ] 弹窗菜单按层级树展示。
- [ ] 父级菜单显示 `无法关联`。
- [ ] 末级菜单显示 `仅关联` 和 `关联并上架`。
- [ ] 已关联菜单展示当前状态和 `取消关联`。

### 11.3 联动效果

- [ ] 点击 `仅关联` 后，列表字段新增对应菜单，状态为未上架。
- [ ] 点击 `关联并上架` 后，列表字段新增对应菜单，状态为已上架。
- [ ] 点击 `取消关联` 后，列表字段移除对应菜单。
- [ ] 多选内容时，操作对所有已选内容生效。

### 11.4 筛选

- [ ] 内容关联多个菜单时，按任一已关联菜单筛选均可命中。
- [ ] 筛选后，列表“关联菜单”字段仍展示该内容全部已关联菜单。

## 12. 风险与注意事项

1. 多菜单关联后，同一内容可能出现在多个菜单页面，需要避免运营误判为重复内容。
2. 内容整体上架状态和菜单维度上架状态需要区分清楚。
3. 父级菜单不允许关联内容。判断标准为“是否存在子菜单”：有子菜单则不可关联，无子菜单则可关联。
4. 如果后续支持菜单节点动态加载，需要处理展开、收起和分页加载。
5. 批量操作时，如果多条内容状态不一致，弹窗状态建议以后增加“部分已关联”状态，本期先按当前选中第一条内容回显。
